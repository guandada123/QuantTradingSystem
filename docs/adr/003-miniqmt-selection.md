# ADR-003: MiniQMT 券商接口选型

**状态**: ✅ 已采纳  
**日期**: 2026-06-05  
**决策者**: QuantTradingSystem 团队

---

## 背景

需要选择一个A股交易接口实现自动化下单。核心需求：

1. **程序化下单**: 代码调用，非 GUI 点击
2. **低延迟**: 从信号生成到订单提交 < 2s
3. **持仓查询**: 实时获取账户持仓和盈亏
4. **成本**: 团队资金有限（¥15,000实盘），接口费用不能过高
5. **合规**: 接口需在监管框架内合法运行

---

## 决策

**采用 同花顺 MiniQMT（通过 xtquant SDK）。**

### 候选方案对比

| 维度 | MiniQMT | XTP (中泰) | 华泰ATP | 手动交易 |
|------|:--:|:--:|:--:|:--:|
| 开户门槛 | 任意券商 | 中泰证券 | 华泰证券 | 任意券商 |
| 程序化下单 | ✅ API | ✅ API | ✅ API | ❌ 人工 |
| 延迟 | 低 (<1s) | 低 | 低 | 高 (>10s) |
| 费用 | 免费 | 免费(有条件) | 免费 | 免费 |
| 文档质量 | 一般 | 较好 | 较好 | N/A |
| 社区支持 | 较大(QMT社区) | 小 | 小 | N/A |
| 模拟交易 | ✅ 内置 | ✅ | ✅ | ❌ |
| Python SDK | ✅ xtquant | ✅ | ✅ | N/A |

### 关键选择因素

**1. xtquant SDK 成熟度**: 同花顺作为A股市场份额第一的交易软件，QMT/MiniQMT 经过多年打磨。xtquant 提供：
   - `xtdata`: 行情数据（K线、分时、Tick）
   - `xttrader`: 交易执行（下单、撤单、查持仓）
   - `xtconstant`: 常量定义（订单类型、价格类型）

**2. 任意券商兼容**: 不同于 XTP（仅中泰）和 ATP（仅华泰），MiniQMT 对接的是同花顺客户端，支持大部分主流券商。
用户当前使用广发证券，无需更换券商。

**3. 模拟交易支持**: MiniQMT 内置模拟交易环境，可在不投入真实资金的条件下验证策略。
这对应本项目 "分阶段集成策略" (ADR-004)。

**4. 社区生态**: 量化交易社区（JoinQuant、BigQuant、掘金）有大量 MiniQMT 教程和示例代码，降低学习成本。

---

## 后果

### ✅ 正面

1. **零接口费用**: 不需要额外的行情或交易 API 订阅
2. **模拟交易验证**: 可先跑模拟盘验证策略再切换实盘
3. **xtquant 功能完备**: 覆盖行情、交易、账户查询全链路

### ❌ 负面

1. **Windows 依赖**: MiniQMT 客户端仅支持 Windows（通过 xtquant 远程连接可部分缓解）
2. **文档不完善**: 官方文档存在遗漏，部分 API 参数需社区试错确认
3. **交易时段限制**: MiniQMT 需在同花顺客户端登录状态下运行，非交易时间无法测试
4. **单客户端限制**: 一个 MiniQMT 实例通常只支持一个账户

### ⚖️ 权衡

相对于完全手动交易（零延迟但无自动化），MiniQMT 在延迟（<1s）和自动化之间取得了最佳平衡。
如果需要更低延迟（<100ms），可考虑 CTP（期货）或直连交易所，但合规成本远超当前项目范围。

---

## 替代方案（被否决）

### XTP (中泰证券)
- **否决原因**: 需要在中泰证券开户，用户不愿更换券商

### 华泰 ATP
- **否决原因**: 同上，限制券商。且华泰ATP对个人投资者有资金门槛

### 远程桌面 + 按键精灵
- **否决原因**: 不靠谱、不稳定、无法通过合规审查

### 完全手动交易
- **否决原因**: 无法实现秒级响应，违背系统设计目标
